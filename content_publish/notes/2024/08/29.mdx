BOMP 및 사업 진행상황 공유

BOMP
- ORDER / MBL / HBL / CONTAINER / PO 매칭 등의 복잡한 액션
	- 오더 상세 화면에서 HBL 하위 Table 뷰 -> 그리드 뷰로 바뀌는거임
	- 여기서 아코디언 UI가 추가돼 하위 개념 설정을 하위 그리드에서 입력하게 하는 것
	- (모달 안에서)
	- 모달 안에서 저장하면 전체 화면에 반영되는 것
- 오더액션
	- As-is Todo 목적: 운영팀 KPI 측정 + 운영 프로세스 가이드 용도
	- 그러나 시간이 지나면서 업무 분담화 등 조직 변화로 원래 목적을 잃어버림
	- 추가로 단계별 값을 입력하는 것과 Progress 간 동기화가 안돼있어 수동으로 전환해줘야하는 불편 존재
	- To-be: Progress(가칭) 변경, 이후 BOMP나 오더상세에서 특정 값 입력/수정 시 자동으로 상태가 동기화됨
	- 과도기 - 완벽한 동기화 전에 '오더통합관리 안에서' Progress 상태 변화를 수동으로 할 수 있게 해야함.
- Fare
	- 대분류 = 비용 관리
	- 마찬가지로 `업체관리 > 화주 > 설정`에서 화주/업체별 비용 컬럼셋 설정할 수 있어야 함
	- 컬럼셋이 GET에 반영되고 열려진 비용 컬럼만 값을 입력할 수 있도록.
	- + 그리드 아래에 탭(?) 기능 필요함(오더관리/비용관리)
프로덕트 솔루션 제공
- 양재를 버리고 똑똑한 솔루션으로 바꾸고 싶은 시장의 니즈가 거대함
- TOMS는 이제 경쟁력이 있는 수준까지 올라왔음
- 종국에는 메일, 엑셀, 카톡 등의 다양한 데이터 출처에서 제품에 맞게 가공하는 중간 과정 없이 AI를 활용해 바로바로 우리 제품에서 데이터를 관리하고, 클릭 몇 번으로도 모든 단계의 작업을 완료할 수 있는 서비스를 제공할 것
- 그 과정에서 현 단계의 우리 제품을 사용하고 싶은 기업들이 있음
- 이들에게 유지보수 없이 다량의 오더를 관리할 수 있는 우리 제품의 기능들을 조합해 솔루션 형태로 제공하고 구독료를 받아낼 수 있다
- 그럼 왜 프로젝트를 분리해서 제공하느냐?
